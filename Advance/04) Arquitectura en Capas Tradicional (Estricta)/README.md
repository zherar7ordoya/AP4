 # CRUD con Arquitectura Tradicional en Capas

**El presente es un modelo básico de un buen diseño de arquitectura de software para una mayor mantenibilidad y menor cohesión.**

Sobre una base de:

- POO (Programación Orientada a Objetos)
- DDD (Domain-Driven Design: Diseño Guiado por el Dominio)
- PEAA (Patterns of Enterprise Application Architecture: Patrón de Arquitectura de Aplicaciones Empresariales)
- Validaciones
- C#
- SQL Server

## Bibliografía

- Patterns of Enterprise Application Architecture (Martin Fowler – 2003)
- Domain-driven design-DDD (Eric Evans -2003)
